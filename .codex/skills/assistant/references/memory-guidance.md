# Assistant Memory Guidance

Preserve durable meaning, not motion. Memory should make future work easier to resume without becoming an archive of ordinary activity.

## Owners

- `agent/ASSISTANT_SETUP.md`: onboarding and thread-setup progress.
- `agent/USER_CONTEXT.md`: durable user profile and operating preferences.
- `people/*.md`: recurring collaborators and evidence-backed relationship context.
- `projects/<project>/README.md`: durable workstream state and open loops.
- `AGENTS.md`: stable routing and operating rules.

## Keep

- stated goals, responsibilities, preferences, and interruption thresholds;
- durable workstreams, decisions, owners, constraints, and source routes;
- important recurring collaborators and why they matter;
- accepted thread topology and recurring-help decisions;
- corrections that should change future behavior.

## Usually Skip

- raw messages, transcripts, and broad source dumps;
- ordinary inbox or thread churn;
- temporary branch, review, or task state;
- people mentioned once without durable relevance;
- weak personality or motivation guesses.

## Evidence And Approval

Separate facts, user statements, and interpretation when the distinction changes future behavior. Keep uncertain interpretations tentative. Ask before writing shared memory, and update the nearest existing canonical file rather than creating an adjacent summary.

Connected sources are optional evidence, not a requirement. Read them only when the user approves or the current request already authorizes the specific read, and only when they materially improve the artifact or decision.

When a correction invalidates durable memory, update the canonical owner after approval and stop using the older interpretation.
